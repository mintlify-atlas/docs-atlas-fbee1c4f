title: Sorting
description: Algorithms for arranging elements in a specific order
---

Sorting algorithms are a set of techniques used to arrange elements, with each algorithm having its own strengths and weaknesses, making it suitable for different scenarios and types of data.

## Sorting Algorithms

| Name                     | Stable? | Time                            | Space       |
| ------------------------ | ------- | ------------------------------- | ----------- |
| [Bubble Sort](./bubble-sort) | Yes     | `O(n^2)`                        | `O(1)`      |
| [Selection Sort](./selection-sort) | No      | `O(n^2)`                        | `O(1)`      |
| [Insertion Sort](./insertion-sort) | Yes     | `O(n^2)`                        | `O(1)`      |
| [Merge Sort](./merge-sort) | Yes     | `O(n log(n))`                   | `O(n)`      |
| [Quick Sort](./quick-sort) | No      | `O(n log(n))` (worst: `O(n^2)`) | `O(log(n))` |
| [Counting Sort](./counting-sort) | Yes     | `O(n + k)`                      | `O(k)`      |
| [Radix Sort](https://www.w3schools.com/dsa/dsa_algo_radixsort.php) | Yes     | `O(nk)`                         | `O(n + k)`  |
| [Heap Sort](https://en.wikipedia.org/wiki/Heapsort) | No      | `O(n log(n))`                   | `O(1)`      |

<Note>
- **Stable**: Equal elements keep their original order.
- **Unstable**: Equal elements might not keep their original order.
</Note>
